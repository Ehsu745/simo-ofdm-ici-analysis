# C 階段筆記：DNN 等化器（進行中）

## 目標
用神經網路取代 ZF/MMSE 等化器，最終在時變通道（ICI）下超越傳統等化。
故事線：傳統單抽頭等化在高都卜勒下失效 → DNN 學習非對角 H 的等化映射 → 量化贏多少。

## 路線決策
- **第一里程碑**：靜態通道 + model-aided（餵通道 d 給網路）。
  理由：任務簡單、3080 數分鐘收斂、先把整條 pipeline 跑通並驗證能追平 MMSE。
- 之後延伸：時變通道（DNN 真正能贏 MMSE 的場景）、data-driven（不餵 d，網路自學通道）。

## 已完成（里程碑 1）

### 架構
- model-aided MLP，逐子載波處理（per-subcarrier）
- 輸入 4 維：[Re(y), Im(y), Re(d), Im(d)]（接收符號 + 通道，拆實虛部）
- 輸出 4 類：QPSK 四象限（**分類**，非回歸）
- 3 層隱藏層 × 64 單元 + ReLU

### 結果（靜態通道，EbN0=10dB，測試集）
| 等化器 | BER |
|--------|-----|
| ZF | 3.47e-2 |
| MMSE | 3.47e-2 |
| DNN | 3.50e-2 |

DNN 追平 MMSE，pipeline 驗證成功。訓練 24s（CPU 容器；3080 更快）。

### 重要觀念
**靜態通道下 DNN「追平」MMSE 是預期中最好結果，非缺陷。**
靜態通道 MMSE 已近最佳線性等化，無非線性結構讓 DNN 發揮。
DNN 真正能贏的場景是時變通道（H 非對角、MMSE 單抽頭假設失效）。
此處追平 = 確認流程正確（不在不該贏時假裝贏）。

## 開發中抓到的 bug（reflection 素材）

**症狀**：MSE loss 正常下降（0.11→0.028），但 BER = 1.0（全錯）。
**矛盾**：指標在動、結果卻全錯 → 網路輸出數值接近目標、但判別象限全錯。
**根因**：用 MSE 回歸做等化是建模錯誤。等化的本質是「判對象限」=分類問題（QPSK 四選一），
不是「逼近數值」=回歸。MSE 下，網路傾向輸出接近 0 的保守值（懲罰較小），
導致判別時全壓到同一象限。
**修正**：改 4 類 cross-entropy 分類。BER 立刻從 1.0 降到 3.5e-2，追平 MMSE。

**教訓**：選對損失函數 = 把問題定式對。這與之前 CFO/ICI 的除錯是同一類智慧——
當兩個本該一致的指標矛盾時（loss 降但 BER 全錯 / 洩漏輕微但 BER 慘烈），
矛盾本身就是定位 bug 的線索。

## 里程碑 2：時變通道 + 1D-CNN（進行中，待 3080 完整訓練）

### 方法選擇依據（主流研究）
查證文獻後確認主流做法是 **CNN**：用卷積網路捕捉 ICI 的「鄰近子載波洩漏」結構。
- 文獻：1D-CNN 在 BER/MSE 上優於 LS/MMSE/FFNN；帶殘差的 CNN 在高都卜勒超越 LMMSE。
- 關鍵洞察：ICI = 帶狀 H 矩陣（鄰近子載波洩漏），CNN 卷積核天生「看鄰居」，
  故能學到單點 MLP 學不到的 ICI 結構。

### 架構
- 輸入：每個中心子載波 + 左右各 W 個鄰居（窗口 2W+1）× 4 特徵 [Re/Im(y), Re/Im(d)]
- 3 層 1D 卷積（kernel=3）+ 全連接頭 → QPSK 4 類分類
- 循環邊界（OFDM 子載波本就循環）

### 待 3080 驗證的關鍵問題（誠實記錄）
CNN 能否超越 MMSE，取決於「鄰近子載波 y 裡殘留多少 MMSE 沒用到的 ICI 資訊」。
三種可能結果都有價值：
- 明顯贏 → 複刻主流方法，最強成果
- 小贏/追平 → 誠實結果，說明簡化設定下 ICI 殘留有限，需更強時變或 pilot 輔助
- 負面結果在研究中同樣站得住，比硬湊「贏」更可信

### 檔案
- `train_cnn.py`  單點 fdTs 的 ZF/MMSE/CNN 對比
- `sweep_fdts.py` 掃多個 fdTs，產三線分岔圖（最終成果）

### 在 3080 上執行
```
# 單點驗證（先確認 CNN 在 fdTs=0.2 贏不贏）
python3 gen_data.py --mode jakes --fdTs 0.2 --out data/jakes02.mat
python3 train_cnn.py --data data/jakes02.mat --W 4 --epochs 25

# 完整掃描（產分岔圖）
python3 sweep_fdts.py --fdts 0 0.05 0.1 0.15 0.2 0.25 --epochs 25
```

## 里程碑 2 完成：CNN 在時變通道超越 MMSE（已驗證）

### 最終結果（3080 實測，每點 20000 符號，25 epoch）

| fdTs | ZF / MMSE | CNN | CNN 相對改善 |
|------|-----------|-----|-------------|
| 0.00 | 3.47e-2 | 3.61e-2 | -4%（略輸，無 ICI） |
| 0.05 | 3.47e-2 | 3.62e-2 | -4%（略輸，無 ICI） |
| 0.10 | 4.55e-2 | 4.59e-2 | ~持平（分岔點） |
| 0.15 | 6.15e-2 | 5.97e-2 | +3% |
| 0.20 | 8.37e-2 | 7.38e-2 | +12% |
| 0.25 | 1.088e-1 | 9.00e-2 | +17% |

### 結論
1. **CNN 的優勢專屬於 ICI 場景**：fdTs≤0.05（無 ICI）時 CNN 略輸 MMSE
   （模型較複雜、微幅過擬合）；這反而證明方法的誠實性——不在不該贏時假裝贏。
2. **分岔點在 fdTs≈0.1**：ICI 變強後 CNN 開始勝出。
3. **改善幅度隨都卜勒擴大**（3%→12%→17%）：ICI 越嚴重，傳統等化越無能，
   CNN 從鄰近子載波窗口撈回的資訊越多，相對優勢越大。

### 理論-方法-數據閉環
- 理論（§3）：ICI = 帶狀 H 矩陣，能量洩漏到鄰近子載波
- 方法：1D-CNN 吃鄰近窗口，卷積核捕捉洩漏結構
- 數據：CNN 優勢隨 ICI 增強而擴大
三者一致，互相印證。

### 完整研究弧線（給備審）

## 里程碑 2 強化：高 SNR 壓力測試（CNN 將 BER 砍半）

### 三組對照實驗結論

**1. epoch 25→50：邊際效益有限**
10dB 下 fdTs=0.4，CNN 從 0.157（25ep）→ 0.140（50ep），僅小幅進步。
→ CNN 在 25 epoch 已近收斂，加 epoch 非主要槓桿。

**2. SNR 是真正的槓桿（關鍵發現）**
EbN0=20dB（高 SNR，ICI 主導誤差）下，CNN 優勢大幅放大：

| fdTs | MMSE (20dB) | CNN (20dB) | 改善 |
|------|-------------|------------|------|
| 0.10 | 1.80e-2 | 1.43e-2 | +21% |
| 0.20 | 5.72e-2 | 3.57e-2 | +38% |
| 0.30 | 1.11e-1 | 5.76e-2 | +48% |
| 0.40 | 1.75e-1 | 8.75e-2 | +50%（BER 砍半） |

**物理解釋**：低 SNR 時雜訊主導，CNN/MMSE 都受限，差距被壓縮；
高 SNR 時雜訊小、ICI 成為唯一誤差來源，而 ICI 正是 CNN 專長、MMSE 盲區。
→ SNR 越高，CNN 優勢越純粹顯現，證明「CNN 贏的正是 ICI 那部分」。

**3. 改善幅度隨 fdTs 擴大**（21%→38%→48%→50%）：
ICI 越強，CNN 越把 MMSE 甩開。CNN 的價值與問題難度正相關。

### 建議主成果圖
用 EbN0=20dB 這組（CNN BER 在 fdTs=0.4 為 MMSE 的一半）作為展示主圖，
最能清楚呈現 CNN 對 ICI 的處理能力。

### 場景誠實標註
fdTs=0.3~0.4 屬「壓力測試場景」（接近高鐵/高速上限或略超），
物理合理、學界有模擬此範圍，但非典型行動通訊條件，報告中應註明。
